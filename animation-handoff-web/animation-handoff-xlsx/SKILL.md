# 美术统筹表 · 带图脚本 → .xlsx（合并版）

**范本**：**美术统筹表范例.xlsx**（旧版列位可能为场景 A–G、道具 J–O 等）。**当前生成脚本**列位见 [reference/README.md](reference/README.md)（场景 **A–F**、**单列间隔**、道具 **H–L**、**单列间隔**、角色 **N–R**；无分镜截图列）。列名与语义与范本一致。勿用 openpyxl 打开范例再 save 丢图，见 reference。

**一条工作流**：读脚本 → 业务规则与填表（本节）→ 写 JSON（[schema.md](schema.md)）→ 执行下方 **生成命令** 产出 `.xlsx`。

## 交付形态（固定）

- **终稿仅为 `.xlsx`**：梳理与三块表内容写入 JSON 后跑生成脚本；**不以 Markdown 多表作为终稿**（中间四类清单可用短表/项目符号便于校对）。

## 输入

用户需提供 **已定稿** 脚本（与 `elementary-animation-storyboard-script` 五列表或原始 Excel 结构一致）。常见为 **带图脚本**：分镜/气氛图**嵌在稿内**（Word、PDF、石墨导出、含内嵌图的分镜 xlsx 等），**不要求**另建图片文件夹。若有 **教学知识表**，可一并提供作动效与环节参考。未定稿时先提示确认。

脚本文件须在 **当前工作区**（或可 Read 的路径）内，Agent 才能打开与解析。

## 复用判定（严格以脚本标记为准）

- **仅当脚本的某一行（或该镜/该行对应单元格）出现以下标记之一时**，方可将该条归入「复用」类，并在统筹表名称上使用 **`-复用` 后缀**（半角连字符，紧贴名称，示例：`街道正面-复用`、`客厅全景-复用`）：
  - 场景：**`*复用场景-…*`**
  - 道具：**`*复用道具-…*`**
  - 人设：**`*复用人设-…*`**
- **不得**仅凭「复用第×讲」「同上景」「沿用」等口语或制片习惯推断为复用；无上述三种标记的，一律按**新增**入表（名称**不加** `-复用`）。若语义明显冲突，在描述内直接写清取舍或并列方案，**仍不**擅自标成复用类；**表内不要使用**「待编导确认」类字样。
- 统筹表 **场景名称 / 道具名称 / 角色名称**：凡属复用类，在从脚本归纳出的主名后统一加 **`-复用`**；脚本里的 **`*复用场景-*` / `*复用道具-*` / `*复用人设-*`** 仅用于**制表前判定与四类清单**，**不要**再写进 **场景描述 / 道具描述 / 角色描述** 正文（与名称列重复、干扰美术阅读）。若另有 **`*复用小框场景-*`**、**`*新增道具-*`** 等名称列**未覆盖**的信息，可在描述里用**自然语言**概括要点，避免再堆叠与名称同义的 `*复用×-*` 原文。

## 带图脚本：先捋「新增 / 复用」再填 JSON

在写 JSON / 生成 xlsx 之前，先通读脚本并输出**四类清单**（可用短表或项目符号；与后文统筹表行一一对应或可去重合并）：

| 类别 | 判定要点 |
| --- | --- |
| **复用场景** | 仅当出现 **`*复用场景-…*`** 且按上节写入名称后缀 **`-复用`**。 |
| **新增场景** | 本讲出现**独立环境/景别**（可与镜号多对一），且**未**出现 `*复用场景-*`。同一环境多镜合并为一行，**不要**为每镜拆成重复场景行（除非脚本刻意区分子区域并命名）。 |
| **复用人设** | 仅当出现 **`*复用人设-…*`**，角色名称加 **`-复用`** 后缀。 |
| **新增人设** | 有造型/出场需求且**未**出现 `*复用人设-*`；仅台词条目而无视觉信息时，角色仍入表，描述写「脚本未写造型【推断】」等，**不要**写「待编导确认」类字样。 |

**道具**：以正文与 **`*复用道具-*`** 提取；**仅**带 `*复用道具-*` 的条目为复用道具，**道具名称**加 **`-复用`**。脚本中的 **「以下复用」** 等仅为**小节标题**，须在道具块保留（与范本一致），但**不替代**逐行 `*复用道具-*` 判定——小节下列项若无 `*复用道具-*`，不按复用类命名，不加 `-复用`。

### 读图（稿内配图）：为何常写「未能解析」、怎么做到「图 + 文」

- **原因**：对 **`.xlsx` / `.docx`** 等，Agent 往往只能稳定读到**文字层**；**内嵌图**不会自动变成一张张可视觉理解的图。若只跑表格文字，统筹表就只能据 **画面描述** 写，容易粗、缺层次。
- **推荐工作流（图 + 文）**  
  1. **先把内嵌图落成独立图片文件**（仍在用户本机，路径给 Agent **Read**）：  
     - **按列对齐脚本分镜（推荐）**：导出锚在「画面」列上的图，并生成行号对照清单：  
       `python3 ~/.cursor/skills/animation-handoff-xlsx/scripts/export_xlsx_column_images.py "/path/to/脚本.xlsx" -o "/path/to/脚本_画面配图" --sheet "工作表名" --column 画面 --header-row 2`  
       输出目录内有 **`manifest.json`**（`excel_row` ↔ 文件名），Agent 按 **Excel 行** 与 **同表「画面描述」列** 对照，再 **Read** 每张 `rowXXX_*.png/jpeg`。  
     - **不区列、全量导出**：`python3 ~/.cursor/skills/animation-handoff-xlsx/scripts/export_xlsx_media.py "/path/to/脚本.xlsx" -o "/path/to/脚本_media"`（`image1.png`… **无行号**，需人工或另建映射时再对齐分镜）。  
  2. **输出目录尽量放在当前 Cursor 工作区**（或可 Read 的路径），否则 Agent 可能读不到文件。  
  3. **写统筹表时**：对每个场景/镜组，**同时**依据 **Read 到的图**与 **脚本画面及剧情上下文**，内化 **画风、建筑、时间光源、构图** 等维度后，用**无【】标签的连贯正文**写入描述列（见下文「表内措辞」）。与图不一致处标 **`[推断]`**；信息不足时在正文内写合理假设或并列方案，**不要**使用「待编导确认」类字样。**交付的 Excel 正文里不要写**「对照 row123」、manifest 路径等内部痕迹。  
  4. **仍无法导出或图为空**：再退回报 **`[稿内配图未能解析，描述据文字]`**，**不要**假装已看图。
- **与「嵌图进统筹表」的区别**：上表是**把图变成可读文件以写好文字列**；若用户要求把脚本分镜图 **嵌进统筹表「脚本截图」列**（场景对应该景主镜、道具/人设优先选**该元素最突出的一行** `rowNNN_*.jpeg`），在 JSON 每条下加 **`配图": { "脚本截图": "文件名" }`**，`meta.配图根目录` 指向导出目录；执行 **`generate_art_coordination_xlsx.py`**（依赖 **openpyxl**，嵌图建议安装 **pillow**）。生成器会**按该列当前列宽缩放图片宽度**，并限制高度、**加高该行行高**，避免浮动图盖住右侧「参考图」或描述列文字。导出图来自整格「画面」列，**以镜为最小单位**；若需更小局部可在后期人工裁切另存再填 **`参考图`**。交付表内**不要**写 row 号、manifest 路径等对接痕迹（文件名不含表内说明即可）。

**脚本截图 / 参考图** 两列：未嵌图时**留空**（不要填「—」等占位符），不编 URL。生成表**已删除「分镜截图」列**，仅保留上述两列图位。

## 典型流程（与用户一致）

1. 用户把 **带图脚本** 放进当前工作区（Word / PDF / 分镜 xlsx / md 等），**不必**另传图片文件夹。  
2. 若为 **含内嵌图的分镜 xlsx**，且需写精确 **场景描述**：**先**用本目录 **`export_xlsx_column_images.py`**（或 `export_xlsx_media.py`）导出到**可 Read 路径**，再 **Read** 图片 + 表内文字对照（见上文「读图」）。  
3. Agent **Read** 脚本文件与（若有）导出图：通读脚本，先输出上文四类梳理，再按 **「读图」「截图三列」** 规则填 **美术统筹主表**（场景 / 道具 / 角色 + 表头 + 统计），并与复用标记、**`-复用`** 命名对齐。  
4. **终稿**：写 JSON（[schema.md](schema.md)），执行生成脚本产出 `.xlsx`（中间 JSON 可用临时文件，用户不需要保留时可删）。

```bash
python3 -m pip install openpyxl
python3 ~/.cursor/skills/animation-handoff-xlsx/scripts/generate_art_coordination_xlsx.py \
  handoff_data.json -o "美术统筹表.xlsx"
```

## 输出 1：美术统筹主表（写入 xlsx）

将下列内容填入 JSON 对应字段并由生成脚本写入表文件（列名不变，字段见 [schema.md](schema.md)）。

### 表头信息区（行式说明即可）

- **编导**（姓名）
- **脚本链接**（如石墨 URL，无则「—」）
- **年级**
- **定制互动色彩参考**（若有；无则「—」）

### 统计行

- **场景数量**、**道具数量**、**角色数量**（表头第 8 行）：**仅统计新增**，即 **场景名称 / 道具名称 / 角色名称** 不以 **`-复用`** 结尾的行；**复用行仍写在明细里，但不计入**上述三个数。生成器会按 JSON 明细**自动写入**这三格，无需在 JSON 里手填 `counts`（填了也会被覆盖为自动计算结果）。

### 三块明细表（列名与顺序固定）

**场景：** 序号 | 背景文件名 | 场景名称 | 场景描述 | 脚本截图 | 参考图  

**道具：** 序号 | 道具名称 | 道具描述 | 脚本截图 | 参考图  

**角色：** 序号 | 角色名称 | 角色描述 | 脚本截图 | 参考图  

- **背景文件名**：脚本未写时可空，勿编造文件名。
- **场景描述 / 道具描述 / 角色描述**：从脚本与画面描述归纳，**避免空泛**（不要只写「可特写/可拉远」）。  
  - **表内措辞（重要）**：**不要**在单元格里写 **「【剧情】【画风】【时间与光】【建筑】【构图】【空间】」** 等**标签式前缀**；剧情阶段、画风、光色、建筑气质、构图层次等仅在制表时**内化思考**，写入表中的应是**连贯、可直接给美术看的书面句**，自然串起上述信息，无需让读者看到分类标签。  
  - **画风与色调（必填向）**：场景为主，道具/人设按需——描述中须给出**画风建议**（如儿童向平涂/线面、软边或硬边漫画 UI 等）与**整体色调、冷暖倾向**；**不要**使用「待编导确认」类字样。  
  - **昼夜与光源**：若脚本或分镜之间存在**强烈昼/夜差异**（或接镜时天色跳变风险），须在描述中写清**天色**（天幕色相、云量感）与**主光/辅光**（月光、日光、路灯、室内底光、魔法光层级），并给出调色或衔接上的统一建议。  
  - **不写编导概括句**：表内**不要**出现「夜袭转折」「强化危机节奏」「四格拼贴推进叙事」等**剧情功能/节奏概括**（制表时自知即可）；叙事作用只通过**光色、构图层次、置景与道具关系**等美术可执行信息体现。  
  - **构图与画面**：**场景描述**须写清景别与空间层次（主景/前中后景、画中画位置如「右上—左—右下」）、建筑与关键置景在画幅中的相对位置、材质与体量；**不写**人物具体动作、对白与情节过程，但可嵌入**情境气氛**（如夜色压迫感、回忆段与现实的冷暖对切），用环境与光线传达而非剧情评语，不复述台词。  
  - **综合叙事的美术总控（场景为主，道具/人设按需）**：结合本镜在叙事中的功能（定调、转折、回忆、教学 UI 等），用**自然语句**点出画风倾向（手绘/扁平/绘本感、线面笔触）、建筑与环境气质、时间与主光源（昼夜、室内外主光、魔法光与情节对齐）等；**按需**取舍，避免散文式复述剧情。  
  - **道具/人设**：外形、体量、颜色、材质、关键纹样；可与场景光色、画风要求一致说明。可写「1.5画幅」等若脚本出现。**若可读配图**：用 **Read** 补全；不足处标 **`[推断]`**。  
  - **复用行**：描述列**不要**以 **`*复用场景-…*`** 等脚本标记开头（名称列已体现复用）。
- **教研确认（范围收紧）**：**场景描述、道具描述**中，**仅当**涉及**知识点表述、课标用语、教学结论准确性**须教研把关时，对相应片段标 **`[待教研确认]`**（或 **`【待教研确认】`** 与全表标注体系统一即可）；其余美术置景、构图、色调、道具外形等非知识类内容**不要**标教研确认。**角色描述**侧若出现课标/知识点表述，同样标教研确认；造型信息不足时在正文内写假设并标 **`[推断]`**，**不要**写「待编导确认」类字样。
- **脚本截图 / 参考图**：无图时**留空**，不要写「—」等占位符，不要虚构链接。
- 脚本出现 **「以下复用」** 类道具区时，在道具块内保留同级 **小节标题**（与范本一致；JSON 用 `{"__section__": "小节标题"}`，见 schema）。

## 输出 2：给动画组（用户需要时）

1. **讲解顺序**（叙事流 + 涉及镜号或行号范围）。
2. **难点镜头**（镜号/行、难点、口头强调一句话）。
3. **Q&A 预备**（3～5 条；口头可说明不确定处，**不要**在统筹表正文套「待编导确认」标签）。

## 与「外置配图路径」的关系

- **默认**：图在脚本里，**不**要求用户在 JSON 里填 `配图` 路径。  
- 仅当用户**明确要求**把单独 png/jpg **嵌进** Excel 单元格时，才用 `配图` + `pillow`（见 [schema.md](schema.md)）。

## 重要：不要破坏用户的大范例文件

- **不要**用 openpyxl **打开**范例再 save（会清空范例内旧内嵌图）。始终生成**新文件**。  
- 仅复制范例：`--template ... --copy-only`（见 [reference/README.md](reference/README.md)）。

## 原则

- 不擅自改台词与 **【n】** 锚点。  
- **`*复用场景-*` / `*复用道具-*` / `*复用人设-*`** 在**读脚本、判定复用**时须与脚本逐字一致；**写入表内描述列时勿重复粘贴**（见上文）。复用行 **名称** 须带 **`-复用`** 后缀（见 [schema.md](schema.md)）。  
- 不新增脚本中未出现的场景/道具；若从剧情推断必增，标 **`[推断]`** 并说明依据。
