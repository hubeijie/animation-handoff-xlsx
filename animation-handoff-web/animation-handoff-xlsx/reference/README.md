# 范本对照说明

**常见诉求**：用户只放入**带图脚本**（图嵌在稿内）。按 [SKILL.md](../SKILL.md) 梳理 **新增/复用场景、道具、人设**，写 **场景描述**（Read 稿内图；无法解析则据文字并标注）；终稿为 **JSON → .xlsx**。**不要求**外置图片夹；嵌图进单元格为可选。

你提供的 **`美术统筹表范例.xlsx`**（例如位于 `~/Downloads/`）为**旧版式**参照；**当前生成脚本**为三栏并列、**无「分镜截图」列**：场景 **A–F**（含脚本截图、参考图）→ **单列空白 G** → 道具 **H–L** → **单列空白 M** → 角色 **N–R**（脚本截图、参考图）；第 1–3 行信息区、第 8 行三项数量（**仅统计名称非 `-复用` 结尾的新增行**）、第 10 行表头；**第 11 行起**为数据行。

**请勿**用 `openpyxl` 打开该范例再 `save` 覆盖：库在保存时会**丢弃工作簿内嵌图片**。生成脚本采用「按范例重排版式」的方式写出新文件。

**新生成表的配图**：在 JSON 各行填写 **`配图`**，键名为 **`脚本截图` / `参考图`**（见 [schema.md](../schema.md)），并安装 **`pillow`**，生成器会把本地图片嵌入到对应两列（场景 **E–F**、道具 **K–L**、角色 **Q–R**）。范例里旧图不会自动带入；需从旧表复制时仍在 Excel 中手工操作。

**占位**：无图、无字时**留空**即可；不要整格只填「—」。生成器会把 JSON 里的「—」等占位串写成**空单元格**。

## 交付与生成（速查）

- **终稿**：固定为 **`.xlsx`**（`handoff_data.json` + `scripts/generate_art_coordination_xlsx.py`），字段见 [schema.md](../schema.md)。
- 中间可保留四类清单便于校对，**不以 Markdown 表作为终稿**。

## 复用判定与名称后缀

- **仅**脚本某行出现 **`*复用场景-…*`**、**`*复用道具-…*`**、**`*复用人设-…*`**（无则不算复用，不加后缀）。
- 统筹表 **场景名称 / 道具名称 / 角色名称**：复用类在主名后加 **`-复用`**（示例：`街道正面-复用`、`客厅全景-复用`）。**描述列不要**再写 **`*复用场景-*` 等脚本标记**（与名称重复）。
- 「以下复用」等仅为**道具块小节标题**，不替代逐行 `*复用道具-*`。

## 带图脚本：四类梳理（先于主表）

| 类别 | 脚本侧信号 |
| --- | --- |
| 复用场景 | `*复用场景-…*` → 名称 `-复用` |
| 新增场景 | 独立环境/景别且**无** `*复用场景-*` |
| 复用人设 | `*复用人设-…*` → 名称 `-复用` |
| 新增人设 | 有造型需求且**无** `*复用人设-*` |
| 复用道具 | `*复用道具-…*` → 名称 `-复用` |

详见 [SKILL.md](../SKILL.md) 正文「复用判定」「带图脚本」节。

## 描述写什么（除构图外）

- **场景描述**（及必要时道具/人设）：在**构图、层次、材质**之外，结合剧情与镜次功能，用**连贯句子**写清画风、建筑气质、时间与主光源等；**不要**出现 **【剧情】【画风】** 等标签，只给美术可读的正文。

## 教研确认（场景 / 道具）

- **场景、道具**：仅**知识点 / 课标用语 / 教学结论**相关片段标 `[待教研确认]`；其余美术类描述不标教研确认。
